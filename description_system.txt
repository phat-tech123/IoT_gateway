ESP1 — Sensor Node

ESP1 is responsible for:

Reading temperature sensor
Publishing telemetry
Receiving LED control commands
Publishing device status
ESP1 Tasks
1. Connect to WiFi

Requirements:

Auto reconnect
Stable WiFi connection
2. Connect to MQTT Broker

Broker configuration:

IP   : 192.168.137.1
Port : 1883
3. Read Temperature Sensor

Read sensor periodically.

Example interval:

Every 2 seconds
4. Publish Telemetry

Topic:

esp32/esp1/telemetry

Example payload:

{
  "temperature": 32.5,
  "timestamp": 1715080000
}
5. Subscribe to LED Commands

Topic:

esp32/esp1/cmd/led

Example payload:

{
  "state": "on"
}

Actions:

"on" → LED ON
"off" → LED OFF
6. Publish Device Status

Topic:

esp32/esp1/status

Example payload:

{
  "status": "online",
  "rssi": -60
}
7. Heartbeat

ESP1 periodically sends heartbeat messages.

Example:

{
  "alive": true
}

Purpose:

Detect disconnected devices
Monitor system health

8. MQTT Last Will (Optional)

If ESP1 disconnects unexpectedly:

esp32/esp1/status = offline


ESP2 — AI Analytics Node

ESP2 is responsible for:

Receiving telemetry from ESP1
Running anomaly detection
Triggering alarms
Publishing AI results
ESP2 Tasks
1. Connect to WiFi

Requirements:

Auto reconnect
Stable WiFi connection
2. Connect to MQTT Broker

Broker configuration:

IP   : 192.168.137.1
Port : 1883
3. Subscribe to ESP1 Telemetry

Topic:

esp32/esp1/telemetry

Example payload:

{
  "temperature": 45
}
4. Run AI / Anomaly Detection

Possible methods:

Threshold detection
Spike detection
Moving average anomaly detection
TinyML inference

Example logic:

if (temperature > 40)
5. Publish Anomaly Results

Topic:

esp32/esp2/anomaly

Example payload:

{
  "anomaly": true,
  "score": 0.91
}
6. Trigger Local Alarm

Possible alarms:

LED blinking
Buzzer activation
Relay trigger

Example:

digitalWrite(ALARM_LED, HIGH);
7. Subscribe to LED Commands

Topic:

esp32/esp2/cmd/led

Allows dashboard control of ESP2 LED.

8. Publish AI Status

Topic:

esp32/esp2/status

Example payload:

{
  "ai_running": true
}

MQTT Topic Structure
Telemetry
esp32/esp1/telemetry
esp32/esp2/anomaly
Device Status
esp32/esp1/status
esp32/esp2/status
Commands
esp32/esp1/cmd/led
esp32/esp2/cmd/led

Python Gateway

The Python Gateway:

Subscribes to local MQTT telemetry
Processes JSON payloads
Forwards data to CoreIoT
Gateway Responsibilities
Subscribe Local MQTT

Topic:

esp32/+/telemetry
Forward to CoreIoT

Topic:

v1/gateway/telemetry
Additional Features (Future)
Offline buffering
Local database
Telegram alerts
Historical analytics
CoreIoT Dashboard

Dashboard features:

Real-time temperature graph
Device online/offline status
LED control buttons
Anomaly alarm notifications
Historical telemetry visualization

        ↓

        ↓

        ↓
